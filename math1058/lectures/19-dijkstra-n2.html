Dijkstra Algorithm - \( \mathcal{O}(n^2)\) version

 

Michael Kenna-Allison

Ian Hawke

Faster version

Observation: The subproblem of finding the arc \( (v,w) \) which minimises:

 

(v,w) \in \argmin_{(i,j) \in \delta^+(S)}\left[Y(i) +l_{ij} \right]

Can be recast as:

(v,w) \in \argmin_{j \in V \backslash S}\left( { \argmin_{(i,j) \in \delta^{-}(j): i \in S} \left[Y(i) +l_{ij} \right]}\right)

The previous result corresponds to:

1. (outer argmin) - Choosing the node \(j \in V \backslash S\) we want to reach.

2. (inner argmin) - Choosing the arc \( (i,j) \) we want to reach node \(j\). 

Key difference

Instead of repeatedly scanning \( \delta^+(S)\) for each node at each iteration, we no longer compute particular path lengths multiple times.

 

This new algorithm is of \( \mathcal{O}(n^2)\), compared to \( \mathcal{O}(n^3)\) of the previous version.

 

The key point is that if, for each \( j \in V \backslash S\) ,we already KNEW the arc \( (i,j) \) which minimises \( Y(i) + l_{ij}\), we can scan the vertices \( j \in V \backslash S\) rather than in \(\delta^+(S) \).

Key idea

When trying to ADD to the blue set \(S\), we compute the path lengths and predecessors to the nodes NOT in \(S\), given by the red edges. We store these values and then update as we continue through each iteration.

Key idea

Difference:

 

If \( j \in S\):  The labels \( Y(i) \) and \( P(i) \) are the length and predecessors of the shortest \( s \to i \) path.

 

If \( j \in V \backslash S\): \( Y(j) \) and \( P(j)\) are the length and predecessors of the current shortest \( s \to j \) path.

 

If a shorter path is found, the labels get updated until all nodes are in the set \(S\). 

 

 

Example

Using \( \mathcal{O}(n^2)\) algorithm find all shortest paths.

First Iteration

Initialisation: 

1. Set \( Y(j) = l_{1,j}\) and \( P(j) = 1\) or \( -\).

Node 3 has minimum \( Y\) so we add it to \(S\) and scan all \( j \in V \backslash S\).

Second Iteration

We have:

  • \(Y(3)+ l_{32} =3 \implies \) label on node 2 is updated.
  • \(Y(3) + l_{35} = 3 \implies \) label on node 5 is updated.

 

 

Node 2 (or 5) has minimum \( Y\) so we add it to \(S\) and scan all \( j \in V \backslash S\).

Third Iteration

We have:

  • \(Y(2)+ l_{24} =4 \implies \) label on node 4 is updated.

 

 

Node 5 has minimum \( Y\) so we add it to \(S\) and scan all \( j \in V \backslash S\).

Fourth Iteration

We have:

  • \(Y(5)+ l_{54} =5 \implies \) label on node 4 is not updated.
  • \(Y(5) + l_{56} = 7 \implies \) label on node 6 is updated

 

 

Node 4 has minimum \( Y\) so we add it to \(S\) and scan all \( j \in V \backslash S\).

Fifth Iteration

We have:

  • \(Y(4)+ l_{46} =7 \implies \) label on node 4 is not updated (same length).

 

 

Node 6 has minimum \( Y\) so we add it to \(S\) and scan all \( j \in V \backslash S\).

Sixth Iteration

No more nodes left, algorithm ends.

This is the same result as the \(\mathcal{O}(n^3)\) version. (note there were two possible shortest paths to reach node 6)
