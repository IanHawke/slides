The problem

Now, let's define the shortest path problem!

Given a graph \( G = (V,A) \) with a non-negative length function \(l^+\), and two nodes \(s\) and \(t\), find the shortest \( s \to t\) path.  

The shortest \( 0 \to 5 \) path is highlighted in bold. We will see how to find this using Dijkstra's algorithm

Shortest Path Problem - length function

The length function tells us the distance along any single arc: equivalently, the distance between any two nodes or vertices when moving in a specific direction.

This need not be the same in each direction, hence the notion of a directed graph.

We can calculate the length of a path via:

l(P) = \sum_{a \in P}l(a)

Or equivalently:

l(P) = \sum_{(i,j) \in P} l_{ij}

Key result

Theorem: Subpath optimality

Let \( P = (i_1,i_2), (i_2,i_3),...,(i_{k-1},i_k)\) be an \( i_1 \to i_k\) shortest path.

For any pair of nodes \( (i_u, i_v) \) visited by \(P\), with \(  u<v\), the subpath \(S\) from \( u \to v\) is the shortest \( u \to v\) path.

Subpath optimality - proof

Proof:

This theorem follows by contradiction. For instance, if \(S\) is NOT the shortest \( u \to v\) path, then there is a new \( u \to v\) path \(S'\) such that \( l(S') < l(S)\).

Therefore, we can define a new \( i_1 \to i_k\) path \(P'\) using the shortcut \(S'\):

P' = P \;\backslash \; S \cup S'

Now, from the positivity of the length function it follows that:

l(P') = l(P) - l(S)+l(S') < l(P)

Hence, \(P\) was not the shortest path and we have a contradiction.

Dijkstra's Theorem

To use Dijkstra's algorithm we need to understand Dijkstra's theorem.

Theorem: Let \( S \supseteq V\) containing the source \(s\). Now, let \( Y(i)\), for all \( i \in S\), be the length of the shortest \( s \to i\) path. Now let:

(v,w) \in \argmin _{(i,j)\; \in\; \delta^+(S)} (Y(i) + l_{ij})

Then, \( \phi = P_{s \to v} \bigcup {(v,w)}\) is the shortest \( s \to w\) path, where \( P_{s \to v}\) is the shortest \( s \to v\) path .

i.e. how to reach another node outside \(S\) based on the shortest paths inside \(S\).

Dijkstra's Theorem - proof

Proof: We show that any other path \( \pi\) to any other vertex \( u \in V \backslash S\) has either the same length as \( \phi\) or a strictly larger one.

 

First, decompose the path \( \pi\) as :

\pi = \pi_1 \cup \{(i,j)\} \cup \pi_2\;

Where:

  1. \( (i,j) \in \delta^+(S)\)
  2. \(\pi_1\) is the shortest \( s \to i\) path
  3. \(\pi_2\) is the shortest \(j \to u\) path

 

Now consider the length of the path \( \pi\):

l(\pi) = l(\pi_1)\;+\ l_{ij}\;+\; l(\pi_2) \;

Dijkstra's Theorem - proof

l(\pi) = l(\pi_1)\;+\ l_{ij}\;+\; l(\pi_2) \;

Recall:

Since we chose \((v,w)\) to minimise \( Y(i) + l_{ij}\) it follows that:

l(\pi_1)\;+\ l_{ij} \geq Y(v) + l_{vw}

Finally, since \( l(\pi_2) \geq 0\):

\begin{aligned} l(\pi)& \geq l(\pi_1) + l_{ij}\\ &\geq Y(v)+ l_{vw} \\ & \geq l(\phi) \; \end{aligned}

Hence \( l(\pi) \geq l(\phi)\) for ALL such paths, so \( \phi\) is the shortest \(s \to w\) path.

Dijkstra's algorithm

Idea? - Apply Dijkstra's theorem iteratively across the whole graph, starting from the source node to find the shortest path to each node in the graph.

How? - We rely on two node labels:

  1. \(Y(i)\) : The length of the shortest \(s \to i\) path
  2. \(P(i)\):  Predecessor of \(i\) in the shortest \(s \to i\) path

We will also need the set of nodes \( \{S\}\) "visited" by the algorithm at each stage.

First steps? - At the beginning \(Y(i)=0\) and \(P(i)\) is undefined - (no node before the source node).

Now, how do we move on?

Dijkstra's algorithm

Next steps:

  1. Find the arc \( (v,w) \in \delta^+(S) \) that minimises \(Y(i) + l_{ij}\): where \( i \in S, j \in V \backslash S\).
  2. Set the node label \( Y(w)= Y(v) + l_{vw}\).
  3. Set the node label \(P(w) = v\).
  4. Set \(S \to S \bigcup \{w\}\).
  5. Find the arc \( (v,w_1) \in \delta^+(S) \) that minimises \(Y(i) + l_{ij}\): where \( i \in S, j \in V \backslash S\).
  6. Continue until ALL the nodes are in \(S\) and have a node label.

Example 1

Find the shortest path from node \(1\) to every other node in the following graph.

  1. \(Y(0) =0\)
  2. \(P(0) = --\)
  3. \( S \cup \{1\}\)

 

First iteration - set starting labels

Second iteration

  1. Scan the cut \(\delta^+(S)\) to obtain:
  2. \(Y(1)+l_{12} =4\) 
  3. \(Y(1)+l_{13} =2\)
  4. Choose arc \(l_{13}\)
  5. Update label on node \(3\).
  6. \( S \cup \{3\}\implies S = \{1,3\}\)

Third iteration

  1. Scan the cut \(\delta^+(S)\) to obtain:
  2. \(Y(1)+l_{12} =4\) 
  3. \(Y(3)+l_{32} =2+1=3\)
  4. \( Y(3)+l_{35} = 2+1 =3\)
  5. Choose arc \(l_{35}\)
  6. Update label on node \(5\).
  7. \( S \cup \{5\}\implies S = \{1,3,5\}\)

Fourth iteration

  1. Scan the cut \(\delta^+(S)\) to obtain:
  2. \(Y(1)+l_{12} =4\) 
  3. \(Y(3)+l_{32} =2+1=3\)
  4. \( Y(5)+l_{54} = 3+2 =5\)
  5. \(Y(5)+ l_{56}=3+4=7\)
  6. Choose arc \(l_{32}\)
  7. Update label on node \(2\).
  8. \( S \cup \{2\}\implies S = \{1,3,5,2\}\)

Fifth iteration

  1. Scan the cut \(\delta^+(S)\) to obtain:
  2. \(Y(2)+l_{24} =3+1=4\) 
  3. \( Y(5)+l_{54} = 3+2 =5\)
  4. \(Y(5)+ l_{56}=3+4=7\)
  5. Choose arc \(l_{24}\)
  6. Update label on node \(4\).
  7. \( S \cup \{4\}\implies S = \{1,3,5,2,4\}\)

Sixth iteration

  1. Scan the cut \(\delta^+(S)\) to obtain:
  2. \(Y(4)+l_{46} =4+3=7\) 
  3. \(Y(5)+ l_{56}=3+4=7\)
  4. Choose arc \(l_{46}\)
  5. Update label on node \(6\).
  6. \( S \cup \{6\}\implies S = \{1,3,5,2,4,6\}\)

Algorithm ends

Key points:

  • All nodes have a label
  • All nodes are in S
  • Shortest paths in the graph are known from node \(1\) to every other node
