LaTeX vs Accessibility

  • Ian Hawke

Notes for the teaching meeting.

PDFs

  • LaTeX output mixes style and content.
  • Typically output to PDF, which (eg) screen readers struggle with:
    • accessible fonts;
    • plots;
    • mathematics.
  • Loss of structure makes it inaccessible.
  • Emma Cliffe has more detail on this, including a talk.
Emma Cliffe's talk is detailed here; the points are a summary.

What works

  • MS Word (recent versions).
  • HTML.

With 50+ person-years of LaTeX material to convert, plus maintenance pain, neither is acceptable.

Instead, convert LaTeX to accessible form.

Accessibility support in technology needs people and cash. Therefore only the big formats really work. However, these aren't appropriate as the base single source of truth,

Options

  • Markdown (RMarkdown, Jupyter).
    • Outputs in all formats;
    • Easy to use;
    • Not starting from LaTeX.
  • Pandoc.
    • Outputs in all formats;
    • Hard to use;
    • Converts between all (relevant) formats.
  • Chirun.
    • Outputs in some formats;
    • Moderately easy to use;
    • Can use LaTeX as base format.

To convert a single document add the chirun package and compile. It will give both HTML and PDF output.

What works?

Not all LaTeX packages or features work with HTML/MathJax.

  • Working: ams, beamer, enumerate, graphicx, ifthen, natbib, many more.
  • Not working: mathtools, some cross-referencing issues, symbols that rely on extra fonts (eg esint, txfonts).
  • Probably not working out of the box: your favourite package.

How to use?

  • Python package.
    • Command line install and build.
    • Rapid iteration and test.
    • Painful to upload to Blackboard.
  • LTI.
    • "Just" upload LaTeX files to Blackboard; builds, adds to course.
    • Slow iteration and test.
    • Needs help from iSolutions.

Summary

  • Chirun is one possible solution for accessibility.
  • Few alternatives.
  • We need to take this more seriously.
